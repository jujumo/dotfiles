= Dotfiles
:toc:
:toc-placement: preamble

Personal dotfiles managed with https://www.chezmoi.io/[chezmoi].

chezmoi only tracks the dotfiles themselves. All installation (packages, Oh My Zsh, the `jumo` theme, chezmoi itself) lives in a standalone `install.sh` bootstrap script.

== Install

One line, on a fresh machine:

[source,shell]
----
sh -c "$(curl -fsLS https://raw.githubusercontent.com/jujumo/dotfiles/main/install.sh)"
----

This installs base packages, Oh My Zsh and chezmoi, then runs `chezmoi init --apply` to lay down the dotfiles (including the `jumo` theme).

To point at a different source repo, set `DOTFILES_REPO`:

[source,shell]
----
DOTFILES_REPO=git@github.com:you/dotfiles.git \
  sh -c "$(curl -fsLS https://raw.githubusercontent.com/jujumo/dotfiles/main/install.sh)"
----

== What gets installed

The `install.sh` script sets up:

* *zsh* and common CLI tools (`nano`, `screen`, `git`, `curl`)
* *Oh My Zsh* (unattended, without generating its own `.zshrc` — chezmoi owns that)
* *chezmoi*, into `~/.local/bin`

The `jumo` theme is no longer downloaded here — chezmoi ships it (see below).

== What chezmoi manages

[cols="1m,3"]
|===
|File |Role

|dot_zshrc
|`~/.zshrc` — sets `ZSH_THEME="jumo"` and the plugin list (`cp gitfast common-aliases rsync debian sudo screen docker`), then sources Oh My Zsh.

|dot_oh-my-zsh/custom/themes/jumo.zsh-theme
|`~/.oh-my-zsh/custom/themes/jumo.zsh-theme` — the `jumo` prompt theme, vendored here so the setup is self-contained.
|===

Files not meant for `$HOME` (`README.adoc`, `CLAUDE.md`, `install.sh`) are excluded via `.chezmoiignore`.

== Day-to-day usage

[source,shell]
----
chezmoi apply              # apply changes to $HOME
chezmoi diff               # preview changes
chezmoi add ~/.someconfig  # track a new file
chezmoi edit ~/.zshrc --apply
----
